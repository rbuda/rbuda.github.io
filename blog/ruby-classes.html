Enumerable#find

Enumerable methods produce highly readable code. They shorten up longer methods that required loops to get a result. The enumerable method we are going to dive deep into is the find method. Let's say you wanted to look at numbers 1 through 3 and find all that were divisible by both 5 and 7. You could do something like this that would count until it reached a number that met our divisible by 5 AND 7 requirement:

for i in 1..300
   if i % 5 == 0 && i % 7 == 0 then
      break
   end
   puts i
end

This would print out 1 through 34 and we would know that 35 is the answer. But this is loooonnnggg and what if I wanted to count up to 300 and find all the numbers that met my criteria? This would be more code added to the loop making it 10+ lines. Here is a simpler way to do what put above but with the enumerable #find:

(1..300).find { |num| num % 5 == 0 and num % 7 == 0 }

This would print 35 for us. That obviously looks cleaner. In the ()'s we put 1..300 meaning including 1 and up to and including 300. What if we wanted to find out ALL the numbers that met our divisible criteria and not just the first? The answer is easier than you think:

(1..300).find.all { |num| num % 5 == 0 and num % 7 == 0 }

This returns 35, 70, 105, 140, 175, 210, 245, 280. Much easier to use the enumerable find to FIND what is in a range.
