Display Inline vs Inline Block

These two both can be applied to different elements within CSS. Below are examples of how they might look on a css file:

div {
display: inline;
 OR
display: inline-block;
}

So what do these do and what are their differences? The inline value is a default for all elements. If the user doesn't tell the code to do anything else than it will be inline. The inline element will accept margin and padding. However, it will only push other elements away horizontally and not vertically. This causes problems when the user wants to stack the elements in a row. Thus bringing in inline-block.

Inline-block allows other elements to sit to the right and left while also accepting margin and padding. Inline-block will also allow the element's height and width to be set while inline doesn't. Inline-block is good to know as it simply allows the user to do more.
